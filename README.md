# Caleb Navigator

A campus navigation web application for Caleb University, built with a **Django REST Framework** backend and a **Vue 3** frontend. It uses **Dijkstra's algorithm** for shortest-path routing, displays interactive maps via **Leaflet.js**, and supports GPS-based location detection via the W3C Browser Geolocation API.

---

## Project Structure

```
caleb-navigator/
├── backend/          # Django REST API
│   ├── config/       # Django settings, URLs, WSGI/ASGI

│   ├── manage.py
│   ├── cert.crt      # HTTPS certificate (auto-generated, not committed)

    │   ├── views/    # MapView, LoginView, RegisterView
    │   ├── stores/   # Pinia stores (auth, navigation)
    │   └── router/   # Vue Router
    ├── public/
    └── index.html
```

## Backend Setup (Django)

### 1. Navigate to the backend folder

```bash
cd backend
```

### 2. Create and activate a virtual environment

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python -m venv venv
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Create the `.env` file

Create a file at `backend/.env` with the following content:

```env
SECRET_KEY=your-secret-key-here
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1
```

> **Note:** Never commit your `.env` file. It is already listed in `.gitignore`.

To generate a secure `SECRET_KEY`, run:

```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

### 5. Apply database migrations

```bash
python manage.py migrate
```

### 6. Seed the campus database

This populates all buildings, waypoints, and path edges for Caleb University:

```bash
python manage.py seed_campus
```

### 7. Create a superuser for the Django admin

```bash
python manage.py createsuperuser
```

### 8. Start the development server (HTTPS)

This project runs over HTTPS so that the browser Geolocation API works correctly for location detection. Use `runserver_plus` instead of the standard `runserver`:

```bash
python manage.py runserver_plus --cert-file cert.crt 0.0.0.0:8000
```

The first time this runs, it generates two certificate files in the `backend/` folder:
- `cert.crt`
- `cert.key`

These are self-signed certificates for local development only.

The API will be available at: **`https://127.0.0.1:8000/`**

> **Important:** Keep this terminal open. Do not close it while using the app.

## Frontend Setup (Vue 3 + Vite)

### 1. Navigate to the frontend folder

```bash
cd frontend
```

### 2. Install dependencies

```bash
npm install
```

### 3. Start the development server

```bash
npm run dev
```

The app will be available at: **`https://localhost:5173/`**

> **Note:** The frontend runs on HTTPS to match the backend. The `vite.config.js` is pre-configured to read the SSL certificate files generated by the backend in step 8 above. Always start the backend first so the certificate files exist before starting the frontend.

## First-Time Browser Setup

Because the project uses self-signed certificates, your browser will show a security warning the first time you open each address. This is normal and safe for local development.

**Do this once after starting both servers:**

1. Open **`https://localhost:8000/api/navigation/buildings/`** in the browser
2. Click **Advanced** → **Proceed to localhost (unsafe)**
3. Open **`https://localhost:5173`**
4. Click **Advanced** → **Proceed to localhost (unsafe)**

After accepting both, the browser remembers and will not ask again.

## Location Detection

This app uses the **W3C Browser Geolocation API** for user positioning. Location accuracy depends on the device:

| Device | Positioning Source | Accuracy | Requires Internet |
|---|---|---|---|
| Smartphone | GPS chip + WiFi | 5–30m | No (GPS) / Yes (WiFi assist) |
| Laptop | WiFi positioning | 30–150m | Yes |

For best results during a demonstration:
1. Go outdoors before opening the app
2. Allow 30–60 seconds for a GPS fix to establish
3. Then move indoors — the cached satellite data improves indoor accuracy

A smoothing algorithm is applied client-side to filter low-quality readings and reduce positional drift.

## API & CORS

The Django backend is configured to allow requests from the Vue dev server on both HTTP and HTTPS:

```python
CORS_ALLOWED_ORIGINS = [
    'http://localhost:5173',
    'http://127.0.0.1:5173',
    'https://localhost:5173',
    'https://127.0.0.1:5173',
]
```

If you change the frontend port, update `CORS_ALLOWED_ORIGINS` in `backend/config/settings.py`.

## Authentication

This project uses **JWT (JSON Web Tokens)** via `djangorestframework-simplejwt`.

| Token         | Lifetime   |
|---------------|------------|
| Access Token  | 30 minutes |
| Refresh Token | 1 day      |

Tokens are stored in `localStorage` and passed as `Bearer <token>` in the `Authorization` header. The Axios instance in `frontend/src/api/index.js` handles token attachment and auto-refresh automatically.

## Database

SQLite is used for development and stored at `backend/db.sqlite3`. No additional database setup is required.

The campus data (buildings, waypoints, path edges) is loaded via the `seed_campus` management command. Re-running this command clears and repopulates all navigation data.
